# ثبات الهوية البصرية — قواعد للتطوير

## الهدف

**البساطة** لفريق المجمع — لا تغيير الشكل مع كل ميزة برمجية.

## مصدر التصميم الوحيد

| الملف | الاستخدام |
|--------|-----------|
| `apps/web/src/styles/theme.css` | ألوان CSS (`--primary` = #1e3a8a) |
| `apps/web/src/app/lib/design-system.ts` | عناوين، بطاقات، قائمة، تنبيهات |
| `apps/web/src/app/components/ui/*` | أزرار shadcn — `variant="default"` و `outline` فقط |

## ممنوع عند إضافة ميزة

- ألوان hex جديدة في الصفحات (`#1e40af` مباشرة في كل ملف)
- زر جانبي جديد في القائمة لكل صفحة فرعية (دمج داخل الصفحة الأب)
- تبديل الوضع الليلي محلياً في صفحة واحدة
- بطاقتان متشابهتان بأزرار غير واضحة («رفع» + «تحميل» بدون شرح)

## مسموح

- منطق برمجي، API، جداول بيانات
- تبويب داخل نفس الصفحة (مثل: قائمة | Excel تحت «إدارة الطلاب»)
- أزرار من `Button` + `ds.btnRound` + `variant` القياسية

## Excel — تدفق واحد واضح

كل الخطوات في **إدارة الطلاب → تبويب ملف Excel**:

1. تحميل **نموذج فارغ**
2. تعبئة على Excel
3. (اختياري) **تصدير القائمة الحالية** — نسخة احتياطية
4. **رفع الملف المملوء**
5. **تطبيق على النظام**

## خارطة المشروع (تطوير شامل لاحقاً)

يُفضّل تجميع المواصفات ثم التنفيذ دفعة واحدة للواجهات — هذا الملف يمنع انحراف التصميم أثناء البرمجة.
